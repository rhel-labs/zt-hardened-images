== Introduction

In this lab, you'll get hands-on with Red Hat Hardened Images. You'll explore the three main categories of images (services, tools, and runtimes), understand the difference between distroless and traditional base images, and build your first containerized applications using these minimal, secure foundations.

= UBI Baseline

Red Hat Hardened Images (RHHI) are a catalog of distroless images designed for secure use: smaller attack surface, non-root by default, rebuilt rapidly when CVEs are patched. These images closely track upstream projects to deliver fixes and features faster.

Red Hat Universal Base Images (UBI) are distribution based images designed for ease of use: easy extendability, broader tool support, backporting for API preservation. These images closely track RHEL to deliver the same packages and lifecycles.

This lab runs the same Flask application on a UBI base, a hardened base, a FIPS-enforcing base, and behind a TLS reverse proxy.  First, lets examine the UBI starting point that many of us work with today.

== The Application

The lab app is `rhhi-demo`, a small Flask web app pre-staged in `~/flask/`.
It simply hashes submitted text using MD5, SHA-256, and SHA-512 with a default message included.

[source,bash,role=execute]
----
ls ~/flask/
----

You'll see the application code (`app.py`) and three Containerfiles: `Containerfile.ubi`, `Containerfile.hardened`, and `Containerfile.fips`.

<.> Installing python tools in standard RHEL fashion

This Containerfile is a conventional approach, and will be the comparison point for the rest of the lab.
We install `pip` to the UBI base image to manage Python packages, then use it to install `flask` and any dependencies. 
We can add our application user, install the app, then finish off the container details with any ports to expose and the runtime command.

== Run the Application

NOTE: The `rhhi-demo:ubi` image has been pre-built

[source,bash,role=execute]
----
podman run -d --rm --name rhhi-ubi -p 8080:8080 rhhi-demo:ubi
----

Open link:http://flask-{guid}.{domain}/crypto-demo[*the demo application*^] in your browser. 

image::ubi-demo.png[]

Feel free to enter some text, and click *Hash*.
All three algorithms — MD5, SHA-256, SHA-512 — return a hex digest of the text.
You can also see the output of the `ENV` variable from the Containerfile in the banner.
This will help identify the source image for the container as we will re-use this URL throughout the lab.

== What's in the Image?

[source,bash,role=execute]
----
podman exec -it rhhi-ubi rpm -qa | wc -l
----

UBI includes packages that cover a broad range of use cases: development of new applications, network services, system init services. 
That breadth is convenient for development but is surface area a production runtime does not need.
